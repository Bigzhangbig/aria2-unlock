# Aria2 一键脚本 (P3TERX 增强版) 二改说明文档

## 1. 项目概述
本项目是对 [P3TERX/aria2.sh](https://github.com/P3TERX/aria2.sh) 的深度解析与二次开发说明。P3TERX 的脚本本身是对原作者 **Toyo (逗比)** 脚本的“增强版”继任，整合了“Aria2 完美配置”方案，极大地提升了国内网络环境下的下载体验。

## 2. 核心修改点 (相对于 Toyo 原版)

### 2.1 内核替换 (Aria2 Pro Core)
- **解锁 16 线程限制**：原版 Aria2 在源码中硬编码了单服务器连接数上限为 16。P3TERX 版本使用了修改过的二进制文件，将 `max-connection-per-server` 的上限提升至 128 甚至无限制。
- **多架构适配**：支持 amd64, i386, arm64, armhf，适配了从高性能服务器到树莓派等多种硬件。

### 2.2 自动化运维脚本集成
- **下载完成后自动清理**：
    - 自动删除 `.aria2` 状态文件。
    - 自动删除已下载完成的 `.torrent` 种子文件。
    - 自动清理下载目录中的空文件夹。
- **BT Tracker 自动更新**：集成了自动获取最新 Tracker 列表的逻辑，并通过 Crontab 定时更新，确保 BT 下载始终保持高速。

### 2.3 Rclone 联动功能
- **自动化流**：支持配置 Rclone，实现“离线下载 -> 自动上传到网盘 (OneDrive/Google Drive)”的完整闭环。

## 3. 进阶优化建议 (配合 aria2-unlock 项目)

为了进一步提升极致环境下的下载效率，建议在二改时结合本仓库 (`aria2-unlock`) 的特性进行以下配置调整：

### 3.1 突破 1K 分段限制
在 `aria2.conf` 中，利用 `aria2-unlock` 内核支持 1K 分段的特性：
```conf
# 即使是极小文件也能触发多线程下载
min-split-size=1K
```

### 3.2 极致并发配置
由于内核已完全解锁限制，在需要抢占带宽时（如低速 S3 存储或网盘直链），可尝试以下配置：
```conf
# 建议在 32-128 之间，视服务器性能而定
max-connection-per-server=64
split=64
```

## 4. 常见问题 (FAQ)
- **Q: 为什么我修改了 `aria2.conf` 还是只有 16 线程？**
  A: 如果你使用的是官方原版二进制，即使配置文件写 64，运行中也会被强制重置为 16。必须使用 `aria2-unlock` 编译出的内核。
- **Q: 高线程会导致封号吗？**
  A: 极高线程 (如 256+) 容易触发百度网盘或 115 的反爬机制。建议日常使用 32-64 线程以保证安全。
